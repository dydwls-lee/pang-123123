# PLAN - Phase별 개발 계획 (초안)

각 phase는 실제로 실행/조작 가능한 SW 상태로 마무리합니다. phase가 끝날 때마다 고객님(사용자)이 직접 실행해보고 피드백을 주시면, 이를 반영한 뒤 다음 phase로 넘어가는 방식으로 진행합니다.

관련 문서: `docs/PRD.md`, `docs/FEATURES/main.md`, `docs/FEATURES/game_rule.md`, `docs/FEATURES/mission1.md`

---

## Phase 1. 메인 화면

**목표**
- 앱 실행 시 메인 화면이 뜬다.
- 타이틀, 시작 버튼, 조작 안내(좌/우 이동, 발사 키), 하이스코어 표시(0 또는 더미 값) UI 구성.
- 시작 버튼을 누르면 게임 화면(빈 화면이라도 OK)으로 전환된다.
- 시작 버튼 아래에 '게임 종료' 버튼 추가
- 배경 하늘색으로 변경

**고객님이 확인할 부분**
- 메인 화면 레이아웃과 텍스트가 이해하기 쉬운지.
- 시작 버튼을 눌렀을 때 화면이 실제로 전환되는지.
- 이 단계에서는 아직 게임 플레이 자체는 없습니다 (화면 전환 여부만 확인).

---

## Phase 2. 플레이어 이동 & 와이어 발사

**목표**
- 게임 화면 하단에 플레이어(캐릭터)가 표시된다.
- 좌/우 키로 플레이어가 이동한다.
- 발사 키를 누르면 위쪽으로 와이어(선)가 나간다.

**고객님이 확인할 부분**
- 캐릭터 이동이 자연스러운지 (속도, 화면 경계 처리).
- 와이어 발사 타이밍과 방향이 의도대로인지 (항상 위쪽 고정 발사).
- 아직 공은 등장하지 않으므로, 조작감 자체에 집중해서 봐주시면 됩니다.

---

## Phase 3. 공 등장 & 물리 이동

**목표**
- 화면에 공이 등장해 벽과 바닥에 튕기며 포물선으로 움직인다.
- 공은 화면 밖으로 빠지지 않고 계속 튕긴다.

**고객님이 확인할 부분**
- 공의 움직임이 자연스러운 포물선인지, 튕기는 각도/속도가 어색하지 않은지.
- 이 단계에서는 아직 와이어와 공이 충돌해도 아무 일도 일어나지 않을 수 있습니다 (공의 움직임 자체만 확인).

---

## Phase 4. 충돌 처리 & HP 시스템

**목표**
- 와이어가 공에 맞으면 공이 사라진다(크기 1단계 고정, 분열 없음).
- 플레이어가 공에 닿으면 HP가 100에서 10씩 감소한다.
- 화면에 현재 HP가 표시된다.

**고객님이 확인할 부분**
- 와이어로 공을 맞췄을 때 정확히 사라지는지 (판정이 너무 빡빡하거나 헐렁하지 않은지).
- 공에 닿았을 때 HP가 잘 깎이는지, 화면에 HP가 잘 보이는지.
- HP가 0이 되었을 때 아직 게임 오버 처리가 없을 수 있습니다 (충돌/HP 반영 자체가 핵심 확인 포인트).

---

## Phase 5. 스테이지 클리어 / 게임 오버 & 제한 시간

**목표**
- 화면의 모든 공을 제거하면 "클리어" 화면이 뜬다.
- 제한 시간이 존재하며, 시간 내 클리어하지 못하면 "실패" 처리된다.
- HP가 0이 되면 "게임 오버" 화면이 뜬다.
- 클리어 시 남은 시간 비례 보너스 점수가 반영된다.

**고객님이 확인할 부분**
- 클리어/실패/게임 오버 조건이 각각 의도한 타이밍에 정확히 발동하는지.
- 제한 시간 표시와 점수(시간 보너스 포함)가 화면에 잘 보이는지.
- 메인 화면으로 돌아가거나 재시작할 수 있는 동선이 있는지.

---

## Phase 6. Mission 1 통합 & 폴리싱

**목표**
- Mission 1의 필드 구성(장애물 없는 단순 필드), 공 개수(1~2개), 느린 속도 등 난이도 설정을 최종 반영.
- 메인 화면 → Mission 1 플레이 → 클리어/게임오버 → 메인 화면 복귀까지 전체 흐름이 끊김 없이 동작.

**고객님이 확인할 부분**
- 처음 접하는 사용자가 튜토리얼처럼 쉽게 클리어할 수 있는 난이도인지 (너무 쉽거나 어렵지 않은지).
- 전체 플레이 흐름(시작 → 플레이 → 종료 → 재시작)이 매끄러운지.
- 이 phase부터는 "완성된 Mission 1"으로 보고 세부 피드백(난이도, 조작감, UI 문구 등)을 자유롭게 주시면 됩니다.
